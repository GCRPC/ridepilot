Admin

System Administrators and Provider Administrators can configure many of the settings for the application via the Admin tab.

Application Settings

System Administrators only

System Administrators can set some settings that are application wide:

1.      Click Application Settings.

2.      Click Edit Application Settings.

3.      Edit the application settings:

§  Password Active Count – The number of passwords the application remembers to avoid reuse.

§  Expire Password After – The number of days after setting a password that a user should be prompted to reset their password.

4.      Click Save to update the application settings.

Providers

System Administrators only

System Administrators can create new Providers:

1.      Click Providers.

2.      Click New Provider.

3.      Enter the Provider settings:

§  Name – Provider name

§  Logo – Provider logo

§  Dispatch enabled – Should the Dispatch tab be displayed?

§  Scheduling – Should the scheduling tabs (Trips, Runs, Trip-Runs, etc.) be displayed?

§  Allow Trip entry from Runs page

§  Fields required for a Run to be considered completed - Which fields need to be completed for a Run to be deemed to have been completed.

4.      Click Save to create the new Provider.

Recurring Driver Compliance Events

System and Provider Administrators only

Each Driver may be required to comply with certain events. For example, they attend annual safety courses. While these can be set on an individual basis via the Drivers tab, recurring events that apply to all of the Provider’s Drivers can be defined under the Recurring Driver Compliance Events Admin functionality.

Display Recurring Driver Compliance Event

To display an existing recurring Driver compliance event:

1.      Click Recurring Driver Compliance Events.

2.      A table is displayed listing all of the current defined Driver compliance events. Click the hyperlink in the Event Name column for the appropriate event.

3.      The details of the Driver compliance event is displayed.

New Recurring Driver Compliance Event

To create a new recurring Driver compliance event:

1.      Click Recurring Driver Compliance Events.

2.      Click New Recurring Event.

3.      A blank recurring Driver compliance event form is displayed.

4.      Enter all of the required information, and click Save.

5.      Assuming all of the entered information is validated, the event will be created.

Please refer to the notice at the top of the page regarding when the event will be created for each Driver.

Edit Recurring Driver Compliance Event

To edit an existing recurring Driver compliance event:

1.      Display the existing recurring Driver compliance event.

2.      Click Edit.

3.      Update the event as necessary and allowed, and click Update.

4.      Assuming all of the edited information is validated, the event will be updated.

Please refer to the notice at the top of the page regarding when the event will be created for each Driver, and when event information is editable.

Delete Recurring Driver Compliance Event

To delete an existing recurring Driver compliance event:

1.      Display the existing recurring Driver compliance event.

2.      Click Delete.

3.      A dialog is displayed asking for the deletion to be confirmed, and whether incomplete Driver events should be deleted. Click Confirm to confirm the deletion.

Lookup Tables

RidePilot uses a number of lookup tables, stored in the database, for populating the various lists used within the application. System Administrators can maintain the values within these lookup tables.

Add new Lookup values

System Administrators only

To add a new value to a lookup table:

1.      Select the Lookup Table from the dropdown list.

2.      Click Add.

3.      Enter the new value in the dialog, and click OK.

Edit existing Lookup values

System Administrators only

To edit the text of an existing lookup table value:

1.      Select the Lookup Table from the dropdown list.

2.      Select the appropriate value from the displayed table.

3.      Click Edit.

4.      Edit the value in the dialog, and click OK.

Delete existing Lookup values

System Administrators only

To delete an existing lookup table value:

1.      Select the Lookup Table from the dropdown list.

2.      Select the appropriate value from the displayed table.

3.      Click Delete.

4.      Click OK in the displayed dialog to confirm the deletion of the lookup table value.

The lookup table value is only “soft deleted”. If a value is deleted by mistake, it can be restored by editing the RidePilot database directly.

Show/Hide Lookup Table values

System and Provider Administrators only

A Provider Administrator can select the lookup table values that should be visible for that Provider:

1.      Select the Lookup Table from the dropdown list.

2.      Select the appropriate value from the displayed table.

3.      Click Hide Value or Show Value, as appropriate, toggle the visibility of the value.

4.      Either way, a dialog is displayed asking you to confirm the change. Click OK to confirm.

Translations

System Administrators only

RidePilot is designed to work in multiple languages, and provide the flexibility to allow displayed text to be edited. This is accomplished through the use of translation keys that define the location of various words and phrases within the user interface, and then having localized versions of those keys for each language. For example, the following table shows a selection of translation keys with the English translations:

Key

English (en)

accommodations

Accommodations

advance_notice_required

Advance Notice Required

age_note

What is your birth year?

all_services

All Services

Translation Key Locations

To display the location of the keys in the user interface:

1.      Click the Tag link in the bottom-right corner of the page.

2.      The [keys] are displayed instead of the words and phrases.

3.      To revert back, click the hyperlink for the required translation language.

Edit Translations

To edit the translations:

1.      Determine for the panel title translation key. The translation keys can be displayed by clicking Tags in the bottom-right of the page. Once determined, revert to the appropriate language.

2.      Display the Admin tab and click Translations to display the Translations page.

3.      Use the Search box to find the appropriate translation key.

4.      Click Add or the hyperlink in the appropriate column.

5.      A page is displayed allowing you to enter/edit the translation for the key.

6.      Click Save to save the translation.

The translations can contain HTML elements.
