Admin

System Administrators and Provider Administrators can configure many of the settings for the application via the Admin tab.

Application Settings

System Administrators only

System Administrators can set some settings that are application wide:

1.      Click Application Settings.

2.      Click Edit Application Settings.

3.      Edit the application settings:

§  Password Active Count – The number of passwords the application remembers to avoid reuse.

§  Expire Password After – The number of days after setting a password that a user should be prompted to reset their password.

4.      Click Save to update the application settings.

Providers

System Administrators only

System Administrators can create new Providers and specify a number of settings that will be specific to each Provider:

1.      Click Providers.

2.      Click New Provider.

3.      Enter the Provider settings:

§  Name – Provider name

§  Logo – Provider logo

§  Dispatch enabled – Should the Dispatch tab be displayed?

§  Scheduling – Should the scheduling tabs (Trips, Runs, Trip-Runs, etc.) be displayed?

§  Allow Trip entry from Runs page

§  Fields required for a Run to be considered completed - Which fields need to be completed for a Run to be deemed to have been completed.

4.      Click Save to create the new Provider.

Recurring Driver Compliance Events

System and Provider Administrators only

Each Driver may be required to comply with certain events on a recurring basis. As an example, they may be required to attend annual safety courses. While these can be set on an individual basis via the Drivers tab, recurring events that apply to all of the Provider’s Drivers can be defined in the Admin tab by clicking Recurring Driver Compliance Events.

Components of a Recurring Driver Compliance Event

Before describing how to create a recurring compliance event for Drivers, it is useful to offer a brief overview of the different components used to define such events:

Event Details: Give the event a name, and enter optional notes about the event.

Scheduling: Specify how the recurring event will be scheduled.

·         Together, the Frequency and Schedule fields let you determine how often the event will occur, in years, months, weeks, or days.

·         The First Due Date field allows you to determine when the recurring event begins and starts recurring. For example, if you set Frequency: 6, Schedule: months, and First Due Date: 2017-01-01, the event will occur on the 1st of January and the 1st of July, starting on January 1st, 2017.

Note that when you save your recurring event, a preview of the next 6 recurrences of the event will appear at the bottom of the Scheduling panel.

Scheduling Options: Check “Use Compliance based scheduling” to schedule the next recurring event based on when a driver has completed the last event. For example, if the frequency of an event was set to six months, then the next recurring event for the driver will be scheduled for six months from the prior completion. If this field is not checked, in contrast, events will be scheduled regularly according to the schedule defined in the Scheduling tab, regardless of whether or when the driver completes previously scheduled events.

New Driver Scheduling: Specify the timing for assigning this recurring event to new drivers. There are several ways of doing this, selectable using the Future Start Rule field.

·         If you select Immediately, a new driver will be assigned this event as soon as possible.

·         If you select On Schedule, events will be assigned to a new driver in accordance with the normal schedule defined in the Scheduling panel. For example, if the event occurs annually in August and a driver starts in October 2017, the next assigned event for that driver would occur in August 2018.

·         Finally, if you select After a Time, you can postpone the first event to some number of days, weeks, months, or years after the new driver’s start date.

Display Recurring Driver Compliance Event

To display an existing recurring Driver compliance event:

1.      Click Recurring Driver Compliance Events.

2.      A table is displayed listing all of the current defined Driver compliance events.

3.      Click the hyperlink in the Event Name column to view the details of that recurring event.

New Recurring Driver Compliance Event

To create a new recurring Driver compliance event:

1.      Click Recurring Driver Compliance Events.

2.      Click New Recurring Event.

3.      A blank recurring Driver compliance event form is displayed.

4.      Enter all of the required information, and click Save.

5.      Assuming all of the entered information is validated, the event will be created.

Please refer to the earlier discussion on the components of a recurring event for more information on when the event will be created for each Driver.

Edit Recurring Driver Compliance Event

To edit an existing recurring Driver compliance event:

1.      Display the existing recurring Driver compliance event.

2.      Click Edit.

3.      Update the event as necessary and allowed, and click Update.

4.      Assuming all of the edited information is validated, the event will be updated.

Note that RidePilot does not allow users to edit fields that will affect the timing of future recurring event schedules—for example, you cannot edit Frequency, First Due Date, Future Start Rule, etc. If you need to change the scheduling rules for a recurring compliance event, you should delete the current version of the recurring event and then recreate a new version with the updated scheduling rules.

Delete Recurring Driver Compliance Event

To delete an existing recurring Driver compliance event:

1.      Display the existing recurring Driver compliance event.

2.      Click Delete.

3.      A dialog is displayed asking for the deletion to be confirmed, and whether incomplete Driver events should be deleted. Click Confirm to confirm the deletion.

Recurring Vehicle Maintenance Compliance Events

System and Provider Administrators only

Each Vehicle may be required to meet certain compliance events. For example, a Vehicle may be required to pass a safety inspection on an annual basis. While these can be set on an individual basis via the Vehicles tab, recurring events that apply to all of a Provider’s Vehicles can be defined in the Admin tab by clicking Recurring Vehicle Maintenance Compliance Events link.

Components of a Recurring Vehicle Maintenance Compliance Event

Before describing how to create a recurring compliance event for Vehicles, it is useful to offer a brief overview of the different components used to define such events:

Event Details: Give the event a name, and enter optional notes about the event.

Scheduling: Configure how the recurring event will be scheduled. You may schedule events based on date or mileage by selecting either option in the Schedule Based On field.

·         When you opt to schedule By Date, the Frequency and Schedule fields let you determine how often the event will occur, in years, months, weeks, or days. The First Due Date field allows you to determine when the recurring event begins and starts recurring. For example, if you set Frequency: 6, Schedule: months, and First Due Date: 2017-01-01, the event will occur on the 1st of January and the 1st of July, starting on January 1st, 2017.

·         If you choose to schedule events By Mileage, simply enter the number of miles between event occurrences into the Mileage Frequency field.

When you save your recurring event, a preview of the next 6 events will appear at the bottom of the Scheduling panel.

Scheduling Options: Check “Use Compliance based scheduling” to schedule events only when the vehicle has completed the last event. If this field is not checked, events will be scheduled regularly according to the schedule defined in the Scheduling tab, regardless of whether or not the vehicle completes the events.

New Vehicle Scheduling: Configure how this recurring event is assigned to a new vehicle. There are several ways of doing this, selectable using the Future Start Rule field.

·         If you select Immediately, a new vehicle will be assigned this event as soon as possible.

·         If you select On Schedule, events will be assigned to the vehicle in accordance with the normal schedule defined in the Scheduling panel. For example, if the event occurs annually in August and the vehicle entered service in October 2017, the next assigned event would occur in August 2018.

·         Finally, if you select After a Time, you can postpone the first event to some number of days, weeks, months, or years after the new vehicle entered service. Note that you cannot select After a Time if you are using mileage-based scheduling.

Lookup Tables

RidePilot uses a number of lookup tables, stored in the database, for populating the various lists used within the application. System Administrators can maintain the values within these lookup tables via the Lookup Tables link in the Admin tab, which opens a Lookup Table screen.

Add new Lookup values

System Administrators only

To add a new value to a lookup table:

1.      Select the Lookup Table that you want to modify from the dropdown list. Once you have made your selection, the rest of the page will be updated to show current entries associated with the selected Lookup Table.

2.      Click Add.

3.      Enter the new value in the dialog, and click OK.

Edit existing Lookup values

System Administrators only

To edit the text of an existing lookup table value:

1.      Select the Lookup Table from the dropdown list.

2.      Select the appropriate value from the table that displays all of the current entries for the Lookup Table.

3.      Click Edit.

4.      Edit the value in the dialog, and click OK.

Delete existing Lookup values

System Administrators only

To delete an existing lookup table value:

1.      Select the Lookup Table from the dropdown list.

2.      Select the appropriate value from the displayed table.

3.      Click Delete.

4.      Click OK in the displayed dialog to confirm the deletion of the lookup table value.

The lookup table value is only “soft deleted”. If a value is deleted by mistake, it can be restored by editing the RidePilot database directly.

Show/Hide Lookup Table values

System and Provider Administrators only

A Provider Administrator can select the lookup table values that should be visible for that Provider:

1.      Select the Lookup Table from the dropdown list.

2.      Select the appropriate value from the displayed table.

3.      Click Hide Value or Show Value, as appropriate, toggle the visibility of the value.

4.      Either way, a dialog is displayed asking you to confirm the change. Click OK to confirm.

Translations

System Administrators only

RidePilot is designed to work in multiple languages, and provide the flexibility to allow displayed text to be edited. This is accomplished through the use of translation keys that define the location of various words and phrases within the user interface, and then having localized versions of those keys for each language. For example, the following table shows a selection of translation keys with the English translations:

Key

English (en)

Accommodations

Accommodations

advance_notice_required

Advance Notice Required

age_note

What is your birth year?

all_services

All Services

Translation Key Locations

To display the location of the keys in the user interface:

1.      Click the Tag link in the bottom-right corner of the page.

2.      The [keys] are displayed instead of the words and phrases.

3.      To revert back, click the hyperlink for the required translation language.

Edit Translations

To edit a translation:

1.      Determine the translation key that you will need to access the translation. The translation keys can be displayed by clicking Tags in the bottom-right of the page. Once determined, revert to the appropriate language.

2.      Display the Admin tab and click Translations to display the Translations page.

3.      Use the Search box to find the appropriate translation key.

4.      Click the hyperlink in the appropriate column.

5.      A page is displayed allowing you to enter/edit the translation for the key.

6.      Click Save to save the translation.

The translations can contain HTML elements.
