Settings

System and Provider Administrators only

Provider Administrators are responsible for maintaining information about the Provider:

·         List of common addresses;

·         Editors and Users for the Provider;

·         Provider-level system configurations; and,

·         Extent of the Provider region and Dispatch view center.

A Provider Administrator can access this information via the Settings tab.

Addresses

All Trips created within the system need to be between defined addresses. These addresses can either be created at the same time as the Trip, or common addresses can be uploaded into the system that then can be used for any Trip.

Search Addresses

To find an existing Address:

1.      In the “Addresses” panel, enter the search term in the “Search Addresses:” textbox and click Search.

2.      All addresses that match the search term are displayed.

Add a new Address

Individual common addresses can be added:

1.      Click Create Address.

2.      An Address dialog is displayed. Enter the details of the new address.

3.      Click Save to save the address.

Edit an existing Address

Existing addresses can be edited. To edit:

1.      Search for the required address.

2.      Click the hyperlink in the “Name” field for the address to edit.

3.      The existing address is shown in an edit form. Edit the address as appropriate.

4.      Click Save to update the address.

Delete an existing Address

To delete an existing address:

1.      Search for the required address.

2.      Click the hyperlink in the “Name” field for the address to edit.

3.      Click the Delete hyperlink in top-right corner of the form.

4.      A confirmation dialog is displayed. Click Confirm to delete the address.

Users

The Users for a Provider can log into RidePilot and view and/or update Customers, Drivers, Trips, and Runs, and schedule Trips depending upon their assigned role.

Create User

To create a new User for the current Provider:

1.      Click Create User at the bottom of the “Users” panel.

2.      The “Create New User” page is displayed. Enter the email address of the new User and set their permission level:

a)      System Admin – they are access all functionality within the application.

b)      Admin – they can maintain the current Provider.

c)      Editor – they can create and maintain Customers, Drivers, Trips and Runs for the current Provider.

d)      User – they can view Customers, Drivers, Trips and Runs for the current Provider.

3.      Click Create User to create the user.

Delete User

To delete an existing User for the current Provider:

1.      Click Delete to the right of the appropriate User in the “Users” panel.

2.      Click OK in the confirmation dialog.

Provider-Level Configuration

Some functionality within the system can be configured at a Provider level.

·         Configure Operating Hours. The daily hours the Provider operates.

·         Dispatch. Should the Dispatch tab be visible?

·         Scheduling. Should users be able to schedule Trips?

·         Trip Times Gap. The default time gap (in minutes) between the Trip pickup time and the drop off time.

·         Fields required for a run to be considered valid. What fields are required for a Run to be considered completed?

·         Reimbursement Rates. Provider-specific program reimbursement rates.

·         Device Pools. Create and manage provider-specific device pools.

About device pools: they are used exclusively for the simple dispatch system that requires an Android app to run on GPS-equipped tablets or smartphones. In the dispatch web interface, devices are grouped into device pools using a tree widget. Each pool has a name and an associated color. The color is used for the pins that mark the location of the device in that pool on a Google map.

·         Region Boundaries. Extent of the Provider region.

·         Dispatch View. The center and zoom level to use for the default Dispatch tab map.

Any of these configuration parameters can be edited.
